# 猜词词库说明

## 数据结构

猜词词库主数据位于：

```text
server/src/modules/guessword/data/lexicon.sqlite
```

当前使用 SQLite 词图谱。词库采用质量优先策略，只收录 2-4 字纯中文词语和成语，不再为了凑数量生成伪词。

主要数据源：

- THUOCL：主来源，提供分类词库和 DF 词频
- jieba：词频辅助，补充通用高频词
- chinese-xinhua：补充成语
- 项目内人工校准关系：保证核心词和常见近义/关联关系稳定存在

当前生成规模：

- 词条：9447 个
- 常用词语/领域词：7647 个
- 成语：1800 个，占比约 19.1%
- 双向关系：约 197700 条
- 数字词：0 个
- 长度异常词：0 个
- SQLite 文件大小：约 20MB

生成命令：

```bash
cd server
npm run build:lexicon
```

数据库包含：

- `terms`：所有词条，所有启用词都可以作为目标词
- `relations`：词与词之间的双向关系和权重
- 索引：按词、启用状态、关系源和关系对查询

源数据会缓存到：

```text
server/src/modules/guessword/data/source-cache/
```

该目录不提交到 Git；提交的是生成后的 `lexicon.sqlite`。

## 入库规则

- 必须是 2-4 个中文字符
- 不允许包含数字、英文或符号
- 优先选择常用词、生活词、领域词和常见成语
- 非成语词需要经过 jieba 词典校验，降低专名和伪词进入目标池的概率
- 成语比例控制在 15%-20%，技术、医疗、金融等专业领域词设置较低上限
- 所有启用词都可以作为目标词
- 后续扩充时应继续追加可靠数据源或人工校准关系，而不是机械拼接词

## 当前评分层级

- 猜中目标词：`100`
- 直接关系：`weight * 100`
- 有限二跳关系：`weight1 * weight2 * 78`
- 同类别词：少量基础分
- 字符重合：仅作为兜底，不再主导分数

## 低质量输入

以下输入会被判为低质量：

- 非中文或超过 8 个中文字符
- 3 个及以上完全相同字符堆叠，例如 `生生生`
- 4 个及以上字符且重复比例过高，例如 `医生生生`

低质量输入会返回 `0` 分，并带 `message` 提示。
