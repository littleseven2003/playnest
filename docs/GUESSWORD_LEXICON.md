# 猜词词库说明

## 数据结构

猜词词库主数据位于：

```text
server/src/modules/guessword/data/lexicon.sqlite
```

当前使用 SQLite 词图谱。词库采用质量优先策略，只收录常用 2-4 字中文词语和成语，不再为了凑数量生成伪词。

当前生成规模：

- 词条：约 420 个
- 双向关系：约 6900 条
- 数字词：0 个
- 长度异常词：0 个

生成命令：

```bash
cd server
npm run build:lexicon
```

数据库包含：

- `terms`：所有词条，所有启用词都可以作为目标词
- `relations`：词与词之间的双向关系和权重
- 索引：按词、启用状态、关系源和关系对查询

## 入库规则

- 必须是 2-4 个中文字符
- 不允许包含数字、英文或符号
- 优先选择常用词、生活词、领域词和常见成语
- 所有启用词都可以作为目标词
- 后续扩充时应继续追加人工整理的词表，而不是机械拼接词

## 当前评分层级

- 猜中目标词：`100`
- 直接关系：`weight * 100`
- 有限二跳关系：`weight1 * weight2 * 78`
- 同类别词：少量基础分
- 字符重合：仅作为兜底，不再主导分数

## 低质量输入

以下输入会被判为低质量：

- 非中文或超过 8 个中文字符
- 3 个及以上完全相同字符堆叠，例如 `生生生`
- 4 个及以上字符且重复比例过高，例如 `医生生生`

低质量输入会返回 `0` 分，并带 `message` 提示。
